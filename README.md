# Laboratorio de Código: Soluciones en Java & Python 🚀

¡Hola! Este repositorio funciona como mi laboratorio personal de desarrollo. En él recopilo diferentes aplicaciones, utilidades y experimentos lógicos que he ido programando para consolidar mis habilidades en **Programación Orientada a Objetos (POO)**, desarrollo de **Interfaces Gráficas de Usuario (GUI)** y estructuras de datos.

---

## 🐍 Proyectos y GUIs en Python

### 🕹️ Sistema de Ventanas Interactivas y Lógica Aleatoria (Tkinter)
* **Archivo:** `ventana_lootbox.py` 
* **Descripción:** Aplicación de escritorio con interfaz gráfica interactiva que gestiona flujos entre múltiples ventanas jerárquicas (`Toplevel`). 
* **Conceptos clave aplicados:**
  * Uso de **Tkinter** para el diseño de interfaces y layouts (`grid`, `place`).
  * Integración de la librería **Pillow (PIL)** para la carga y renderizado dinámico de imágenes.
  * Control de eventos, funciones lambda para el paso de parámetros y manejo de excepciones (`try-except-finally`) para evitar la duplicidad de procesos en segundo plano.

---

## ☕ Desarrollo y Lógica Avanzada en Java

### 📋 Control Kids - Sistema de Gestión CRUD (Consola)
* **Archivo:** `control.java`
* **Descripción:** Aplicación orientada a objetos para el registro, lectura y modificación en tiempo real de perfiles de usuario (simulación de base de datos local para control de registros de menores).
* **Conceptos clave aplicados:**
  * **Encapsulamiento estricto (POO):** Creación de la clase `Niño` utilizando atributos privados protegidos mediante métodos de acceso públicos (`getters` y `setters`).
  * **Colecciones dinámicas:** Implementación de un `ArrayList<Niño>` para la manipulación, búsqueda y actualización de objetos en memoria.
  * Interacción robusta por consola mediante flujos de entrada de datos (`Scanner`).

### 🧮 Evolución de Estructuras y Modularización (Calculadoras)
* **Archivos:** `calculadora.java` y `Primer.java`
* **Descripción:** Dos versiones de software de cálculo que demuestran mi progreso en arquitectura de software limpia.
* **Conceptos clave aplicados:**
  * Refactorización de código secuencial hacia un modelo modular basado en métodos específicos (`suma`, `rest`, `mult`, `divi`).
  * Control de flujos continuos mediante bucles condicionales (`while`) e interactividad reutilizable.
